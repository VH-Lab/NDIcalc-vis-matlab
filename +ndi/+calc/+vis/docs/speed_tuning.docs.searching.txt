By default, this ndi.calc.vis.speed object looks for stimulus_tuningcurve documents that have 'spatial_frequency' and 'temporal_frequency' as the independent_variable_label fields of the stimulus_tuningcurve with no other independent_variable_label entries. That is, this calculation object searches for stimulus_tuningcurve documents that have variation in both spatial and temporal frequency values and nothing more.

If one wished to restrict this further, one might add an additional condition to the default query.

The parameters structure returned by default_search_for_input_parameters() also contains an 'input_parameters' field that controls the calculation itself.

input_parameters:
	min_xi: (double, default 0) The lower bound for the speed index (xi) parameter during the fit.
	max_xi: (double, default 1) The upper bound for the speed index (xi) parameter during the fit.

Example:
% Only perform the operation on the element called myelement

DQ = c.default_search_for_input_parameters();
DQ.query = DQ.query & ndi.query('','depends_on','element_id',myelement);
c.run('Replace',DQ);


Example:
% Run with custom xi constraints (default is min_xi=0, max_xi=1)

cc = ndi.calc.vis.speed_tuning(S);
parameters = cc.default_search_for_input_parameters();
parameters.input_parameters.min_xi = 0;
parameters.input_parameters.max_xi = 1;
docs = cc.run('Replace',parameters);


Example:
% Run with extended xi constraints

cc = ndi.calc.vis.speed_tuning(S);
parameters = cc.default_search_for_input_parameters();
parameters.input_parameters.min_xi = -4;
parameters.input_parameters.max_xi = 4;
docs = cc.run('Replace',parameters);
